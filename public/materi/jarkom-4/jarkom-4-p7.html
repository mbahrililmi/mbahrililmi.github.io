2 Teori Dasar
🖧 Konsep VLAN ▼

VLANVirtual LAN: pemisahan logis jaringan tanpa perlu kabel fisik berbeda (Virtual Local Area Network) adalah teknologi yang memungkinkan segmentasi jaringan secara logis tanpa memerlukan perangkat keras tambahan.

Manfaat VLAN:

  • Keamanan: Memisahkan traffic antar departemen (HR tidak bisa akses R&D)
  • Efisiensi: Mengurangi broadcast domain - broadcast tidak menyebar ke semua port
  • Fleksibilitas: User dapat dipindahkan antar VLAN tanpa mengubah kabel fisik
  • Skalabilitas: Satu switch fisik bisa jadi banyak switch virtual
💡 Penting: VLAN bekerja di Layer 2. Komunikasi antar VLAN memerlukan perangkat Layer 3 (router atau Layer 3 switch). Ini yang disebut Inter-VLAN Routing.
🏷️ IEEE 802.1Q VLAN Tagging ▼

Standard IEEE 802.1Q mendefinisikan cara menambahkan tag VLAN ke frame Ethernet:

[Ethernet Header] [802.1Q Tag (4 byte)] [Payload] [FCS] └─ TPID: 0x8100 └─ PCP: 3 bit (Priority) └─ DEI: 1 bit (Drop Eligible) └─ VID: 12 bit (VLAN ID) = 0-4095 (VLAN yang valid: 1-4094)

VLAN ID (VID):

  • Range: 1 - 4094
  • VLAN 1: Default VLAN (tidak disarankan untuk data)
  • VLAN 0 dan 4095: Reserved

🌉 Bridging di MikroTik ▼

Bridge di MikroTik menggabungkan beberapa interface menjadi satu domain Layer 2. Mirip dengan switch virtual yang dikelola oleh RouterOS.

Cara kerja Bridge:

  • Membangun MAC Address Table dari source MAC frame yang diterima
  • Meneruskan frame berdasarkan destination MAC
  • Mem-flood frame ke semua port jika destination MAC tidak dikenal

Penggunaan Bridge:

  • Menghubungkan beberapa interface dalam satu broadcast domain
  • Dasar untuk konfigurasi VLAN dengan Bridge VLAN Filtering
  • Transparent bridging untuk melewatkan traffic Layer 2
⚙️ Bridge VLAN Filtering (RouterOS v7) ▼

RouterOS v7 menggunakan Bridge VLAN Filtering sebagai cara utama konfigurasi VLAN. Ini lebih efisien karena menggunakan hardware offloading.

Konsep utama:

  • pvid (Port VLAN ID): VLAN ID default untuk frame untagged yang masuk ke port ini
  • tagged: Port ini menerima/mengirim frame dengan tag VLAN tertentu
  • untagged: Port ini menerima/mengirim frame tanpa tag (access port)
⚠️ KRITIS: Aktifkan vlan-filtering=yes di bridge SETELAH semua konfigurasi VLAN selesai. Mengaktifkan sebelum konfigurasi lengkap dapat memutus koneksi ke router!

[PC1]──ether2──[MikroTik]──ether3──[PC2] (Bridge br-lan) IP Bridge: 192.168.1.1/24 PC1: 192.168.1.2/24, GW 192.168.1.1 PC2: 192.168.1.3/24, GW 192.168.1.1
💡 Skenario: Jadikan MikroTik sebagai switch dengan menggabungkan ether2 dan ether3 ke dalam satu bridge. PC1 dan PC2 akan berada di subnet yang sama.
1
/interface bridge add name=br-lan comment="Bridge LAN"
2
/interface bridge port add bridge=br-lan interface=ether2 /interface bridge port add bridge=br-lan interface=ether3
3
/ip address add address=192.168.1.1/24 interface=br-lan
4
/interface bridge port print /interface bridge host print # MAC address PC1 dan PC2 harus muncul di host table

5 Lab 30: Inter-VLAN Routing (Router-on-a-Stick)
[PC-HRD (192.168.10.2)]──VLAN10──┐ ├──[MikroTik Router]──ether1──[Trunk]──[Switch] [PC-Lab (192.168.20.2)]──VLAN20──┘ Router-on-a-Stick: vlan10 interface: 192.168.10.1/24 (gateway VLAN 10) vlan20 interface: 192.168.20.1/24 (gateway VLAN 20) Traffic antar VLAN melewati router ini
💡 Router-on-a-Stick: Metode Inter-VLAN routing menggunakan satu physical interface pada router yang terhubung ke trunk port switch. Router membuat sub-interface (virtual interface) untuk setiap VLAN.
