2 Teori Dasar
🧱 4 Tipe Route di MikroTik ▼

MikroTik RouterOS mendukung empat tipe route yang memiliki perilaku berbeda saat menerima paket:

1. Unicast (Default)

Paket diteruskan ke gatewayPintu keluar jaringan menuju jaringan lain yang ditentukan. Ini adalah tipe route standar untuk forwarding paket normal.

Flag: A S | Respon ping: Reply normal

2. Blackhole

Paket di-drop secara diam-diam tanpa mengirim ICMP error balik ke pengirim. Pengirim tidak tahu paketnya dibuang.

Flag: A SB | Respon ping: Timeout (no reply)

3. Prohibit

Paket di-drop dan router mengirim ICMP Type 3 Code 13 (Communication Administratively Prohibited) ke pengirim.

Flag: A SP | Respon ping: "Communication administratively prohibited"

4. Unreachable

Paket di-drop dan router mengirim ICMP Type 3 Code 1 (Host Unreachable) ke pengirim.

Flag: A SU | Respon ping: "Host unreachable"

📡 ICMP Type 3 - Destination Unreachable ▼

ICMP Type 3 adalah pesan error yang dikirim router/host ketika paket tidak dapat diteruskan. Setiap code menjelaskan alasan yang berbeda:

CodeNamaPenyebab
0Net UnreachableJaringan tujuan tidak bisa dijangkau
1Host UnreachableHost tujuan tidak bisa dijangkau (tipe: unreachable)
2Protocol UnreachableProtokol tidak didukung host tujuan
3Port UnreachablePort tidak tersedia di host tujuan
13Communication Administratively ProhibitedDiblokir oleh kebijakan admin (tipe: prohibit)
⏱ TTL (Time to Live) ▼

TTL (Time to Live) adalah nilai dalam header IP yang menentukan berapa banyak hop (router) yang dapat dilalui paket sebelum dibuang. Setiap router yang memproses paket akan mengurangi TTL sebesar 1.

Nilai TTL Default:

  • Linux/MikroTik: TTL = 64
  • Windows: TTL = 128
  • Cisco IOS: TTL = 255

Jika TTL mencapai 0, router mengirim ICMP Type 11 (Time Exceeded) dan membuang paket. Inilah yang digunakan oleh traceroute untuk memetakan jalur.

💡 Kegunaan mengubah TTL: Mengubah TTL dapat digunakan untuk membatasi "jarak tempuh" paket, atau untuk menyembunyikan berapa banyak hop di jaringan internal Anda dari pengguna eksternal.
🔨 Mangle - Packet Marking ▼

Mangle adalah tabel di firewall MikroTik yang digunakan untuk memanipulasi paket. Berbeda dari filter (yang hanya accept/drop), mangle dapat:

  • packet-mark: Menandai paket untuk diproses QoS
  • connection-mark: Menandai seluruh koneksi
  • routing-mark: Menandai paket untuk routing policy
  • change-ttl: Mengubah nilai TTL paket
  • change-mss: Mengubah Maximum Segment Size

Chain Mangle:

  • prerouting: Sebelum routing decision (untuk traffic masuk ke router)
  • postrouting: Setelah routing decision (untuk traffic keluar dari router)
  • forward: Traffic yang melewati router (tidak ditujukan untuk router)
  • input: Traffic yang ditujukan ke router itu sendiri
  • output: Traffic yang berasal dari router itu sendiri
3 Lab 12: Unicast Route
💡 Unicast adalah tipe route default - paket diteruskan ke gateway. Ini adalah route normal yang sudah dipelajari di P1.
1
# Unicast adalah default - tidak perlu parameter tambahan /ip route add dst-address=192.168.100.0/24 gateway=10.10.10.2 # Atau explicit: /ip route add dst-address=192.168.100.0/24 gateway=10.10.10.2 type=unicast
2
/ip route print # Output: A S 192.168.100.0/24 10.10.10.2 1

4 Lab 13: Blackhole Route
⚠️ Blackhole: Paket di-drop DIAM-DIAM. Pengirim mendapat timeout dan tidak tahu apakah tujuan ada atau tidak. Berguna untuk membuang traffic tertentu tanpa memberikan informasi kepada penyerang.
1
/ip route add dst-address=192.168.200.0/24 type=blackhole
2
/ip route print # Output: A SB 192.168.200.0/24 (blackhole)
3
5 Lab 14: Prohibit Route
💡 Prohibit: Paket di-drop dan pengirim mendapat ICMP Type 3 Code 13 "Communication administratively prohibited". Pengirim tahu traffic diblokir oleh kebijakan admin.
1
/ip route add dst-address=192.168.201.0/24 type=prohibit
2
/ip route print # Output: A SP 192.168.201.0/24
3
6 Lab 15: Unreachable Route
💡 Unreachable: Paket di-drop dan pengirim mendapat ICMP Type 3 Code 1 "Host unreachable". Memberikan kesan bahwa host tersebut tidak bisa dijangkau.
1
/ip route add dst-address=192.168.202.0/24 type=unreachable
2
/ip route print # Output: A SU 192.168.202.0/24

7 Lab 16: Routing Mark PBR (Lanjutan)
💡 Lab 16 menggunakan topologi yang sama dengan Lab 12 (P3). Ini adalah implementasi Policy Based Routing menggunakan Routing Mark yang lebih detail.
PC1 (192.168.1.0/24)────[R1]────ether1:12.12.12.0/30────[R2] PC2 (192.168.10.0/24)──┘ └──ether2:21.21.21.0/30────[R2] Kebijakan PBR: PC1 → mark "ISP1-mark" → route via 12.12.12.2 PC2 → mark "ISP2-mark" → route via 21.21.21.2
1
/ip firewall mangle add chain=prerouting src-address=192.168.1.0/24 \ action=mark-routing new-routing-mark=ISP1-mark passthrough=yes add chain=prerouting src-address=192.168.10.0/24 \ action=mark-routing new-routing-mark=ISP2-mark passthrough=yes
2
/ip route add dst-address=0.0.0.0/0 gateway=12.12.12.2 \ routing-mark=ISP1-mark
3
/ip route add dst-address=0.0.0.0/0 gateway=21.21.21.2 \ routing-mark=ISP2-mark
8 Lab 17: Change TTL dengan Mangle
⚠️ Use Case: Mengubah TTL paket yang meninggalkan router. Misalnya: mengubah TTL menjadi 10 agar paket dari jaringan internal tidak dapat melakukan traceroute melebihi 10 hop ke luar.
1
2
/ip firewall mangle add chain=postrouting \ out-interface=ether1 \ action=change-ttl \ new-ttl=set:10 \ comment="Ubah TTL ke 10 untuk traffic keluar"
