2 Teori BGP
🌎 Dasar BGP ▼

BGPBorder Gateway Protocol: protokol routing antar jaringan besar/ISP (Border Gateway Protocol) adalah protokol routing tipe Path Vector yang digunakan untuk routing antar Autonomous System (AS). BGP adalah "bahasa internet" yang digunakan antar ISP.

  • RFC: 4271
  • Transport: TCP Port 179
  • Tipe: Path Vector (berbeda dari Link State dan Distance Vector)
  • AD eBGP: 20
  • AD iBGP: 200
  • Peering: Manual (tidak otomatis seperti OSPF)
💡 Path Vector: BGP membawa informasi tentang jalur AS (AS-Path) yang harus dilalui untuk mencapai suatu prefix. Ini mencegah routing loop antar AS.
🏢 Autonomous System Number (ASN) ▼

Autonomous System (AS) adalah sekelompok router di bawah satu administrasi yang menggunakan routing policy yang sama ke internet.

📝 Atribut BGP & Best Path Selection ▼

BGP memilih satu best path dari beberapa route ke prefix yang sama menggunakan atribut-atribut berikut (urutan prioritas):

  1. Weight (Cisco-specific): Tertinggi dipilih
  2. Local Preference: Tertinggi dipilih (default=100). Digunakan dalam AS untuk preferensi exit point.
  3. Locally Originated: Route yang di-originate lokal (network/redistribute) lebih disukai
  4. AS-Path Length: Lebih pendek lebih disukai
  5. Origin: IGP (i) > EGP (e) > Incomplete (?)
  6. MED (Multi-Exit Discriminator): Terendah dipilih. Digunakan untuk preferensi entry point dari AS tetangga.
  7. eBGP lebih disukai dari iBGP
  8. IGP metric ke next-hop: Terendah dipilih
🔄 BGP State Machine ▼

BGP melalui beberapa tahap sebelum mencapai state Established:

Idle → Connect → Active → OpenSent → OpenConfirm → Established
  • Idle: BGP tidak aktif / menunggu start event
  • Connect: Mencoba membuat koneksi TCP ke peer
  • Active: TCP connection gagal, mencoba lagi. Jika terus di sini, cek konektivitas ke peer.
  • OpenSent: TCP berhasil, mengirim OPEN message
  • OpenConfirm: OPEN diterima, menunggu Keepalive/Notification
  • Established: Peering berhasil! Siap bertukar routing info

4 Lab 26: iBGP dan Next-Hop-Self
[LAN1]──[R1]──ether1──[R2]──ether2──[R3]──[LAN3] AS 65001 eBGP iBGP iBGP AS 65001 ↑ AS 65002 (R2 adalah ABR) R2 dan R3 dalam AS 65002 (iBGP)
⚠️ Masalah Next-Hop iBGP: Ketika R2 menerima route dari R1 (eBGP) dan mengiklankannya ke R3 (iBGP), next-hop tetap IP R1 (10.10.10.1). R3 mungkin tidak tahu cara mencapai 10.10.10.1. Solusi: nexthop-choice=force-self (atau next-hop-self) agar R2 mengganti next-hop dengan IP-nya sendiri.
1
/routing bgp template add name=iBGP-template as=65002 \ router-id=2.2.2.2 /routing bgp connection add name=to-R3 \ template=iBGP-template \ remote.address=192.168.100.3 \ remote.as=65002 \ local.address=192.168.100.2 \ nexthop-choice=force-self
2
/routing bgp template add name=iBGP-template as=65002 \ router-id=3.3.3.3 /routing bgp connection add name=to-R2 \ template=iBGP-template \ remote.address=192.168.100.2 \ remote.as=65002 \ local.address=192.168.100.3
