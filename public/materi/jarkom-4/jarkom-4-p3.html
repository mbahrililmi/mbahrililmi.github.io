2 Teori Dasar
🧭 Routing Policy ▼

Routing Policy adalah mekanisme pengambilan keputusan routing yang lebih kompleks dari sekadar melihat destination address. Policy routing mempertimbangkan berbagai kriteria:

  • Source address: Dari jaringan mana paket berasal
  • Destination address: Ke mana paket dituju
  • Interface: Dari interface mana paket masuk
  • ToS (Type of Service): Jenis layanan dalam header IP
  • Mangle marking: Label yang diberikan oleh aturan firewall mangle
💡 Perbedaan dari Static Routing Biasa: Static routing biasa hanya melihat destination address. Routing policy dapat membuat keputusan berdasarkan siapa pengirimnya, bukan hanya ke mana tujuannya.
📍 Policy Based Routing (PBR) ▼

Policy Based Routing (PBR) memungkinkan administrator untuk menentukan jalur routing berdasarkan kebijakan tertentu, bukan hanya tabel routing standar.

Contoh use case PBR:

  • Departemen IT (192.168.1.0/24) selalu keluar via ISP1 (dedicated/cepat)
  • Departemen Umum (192.168.10.0/24) keluar via ISP2 (lebih murah)
  • Traffic VoIP diarahkan ke jalur dengan latensi rendah
  • Traffic backup/download diarahkan ke jalur bandwidthKapasitas maksimum jalur data, diukur dalam Mbps/Gbps besar

PBR di MikroTik diimplementasikan menggunakan kombinasi Mangle + Routing Table + Route dengan routing-mark.

🏷️ Routing Mark (Mangle) ▼

Routing Mark adalah label yang diberikan kepada paket melalui aturan firewall mangle. Paket yang memiliki routing-mark tertentu akan mencari route di tabel routing yang sesuai dengan mark tersebut.

Alur kerja PBR dengan Mangle:

  1. Paket masuk dari PC1 (192.168.1.x) → Mangle mencocokkan src-address
  2. Mangle memberikan routing-mark "ISP1-traffic" pada paket
  3. Router mencari route di tabel routing yang memiliki routing-mark "ISP1-traffic"
  4. Paket diteruskan sesuai route di tabel tersebut
# Contoh struktur mangle rule /ip firewall mangle add chain=prerouting \ src-address=192.168.1.0/24 \ action=mark-routing \ new-routing-mark=ISP1-traffic \ passthrough=yes

🌐 NAT Masquerade ▼

NATNetwork Address Translation: menerjemahkan IP privat ke IP publik Masquerade adalah tipe NAT dinamis yang secara otomatis mengganti source IP paket dari IP private ke IP public interface yang digunakan.

Pada konfigurasi multi-WAN dengan Routing Policy, diperlukan masquerade di setiap interface WAN agar return traffic dapat diterima dengan benar.

# NAT Masquerade untuk ether1 (WAN ke ISP1) /ip firewall nat add chain=srcnat action=masquerade out-interface=ether1 # NAT Masquerade untuk ether2 (WAN ke ISP2) /ip firewall nat add chain=srcnat action=masquerade out-interface=ether2
⚠️ Penting: Tanpa masquerade yang benar pada setiap WAN interface, traffic yang keluar via ISP2 tidak akan mendapatkan reply karena source IP private tidak dikenali di internet.

Konfigurasi Mangle di R1 untuk marking traffic:

1
/ip firewall mangle add chain=prerouting \ src-address=192.168.1.0/24 \ action=mark-routing \ new-routing-mark=ke-ISP1 \ passthrough=yes \ comment="Traffic PC1 via ISP1"
2
/ip firewall mangle add chain=prerouting \ src-address=192.168.10.0/24 \ action=mark-routing \ new-routing-mark=ke-ISP2 \ passthrough=yes \ comment="Traffic PC2 via ISP2"
3
/ip firewall mangle print

Konfigurasi Route dengan routing-mark di R1:

1
/ip route add dst-address=0.0.0.0/0 gateway=12.12.12.2 /ip route add dst-address=200.200.200.0/24 gateway=12.12.12.2 /ip route add dst-address=200.200.100.0/24 gateway=21.21.21.2
2
/ip route add dst-address=0.0.0.0/0 gateway=12.12.12.2 \ routing-mark=ke-ISP1
3
/ip route add dst-address=0.0.0.0/0 gateway=21.21.21.2 \ routing-mark=ke-ISP2

Konfigurasi NAT Masquerade di R1:

1
/ip firewall nat add chain=srcnat action=masquerade \ out-interface=ether1 comment="NAT ke ISP1"
2
/ip firewall nat add chain=srcnat action=masquerade \ out-interface=ether2 comment="NAT ke ISP2"
3
/ip firewall nat print

Verifikasi Routing Policy

1
# Dari PC1: tracert 200.200.200.1 # Hop 1: 192.168.1.1 (R1) # Hop 2: 12.12.12.2 (R2 via ether1 - ISP1) ← BENAR
2
# Dari PC2: tracert 200.200.100.1 # Hop 1: 192.168.10.1 (R1) # Hop 2: 21.21.21.2 (R2 via ether2 - ISP2) ← BENAR
3
/ip firewall mangle print stats # Lihat kolom "packets" dan "bytes" bertambah sesuai traffic
✅ Tips Troubleshooting: Jika routing policy tidak bekerja, cek urutan mangle rules (/ip firewall mangle print). Aturan diproses dari atas ke bawah. Juga pastikan passthrough=yes agar paket bisa diproses aturan berikutnya setelah di-mark.
