2 Teori OSPF
📖 Dasar OSPF ▼

OSPFOpen Shortest Path First: protokol routing dinamis yang mencari jalur terpendek (Open Shortest Path First) adalah protokol routing dinamis berbasis Link State yang menggunakan algoritma Dijkstra (SPF - Shortest Path First) untuk menghitung jalur terpendek.

  • RFC: 2328 (OSPFv2), 5340 (OSPFv3 untuk IPv6)
  • Protokol IP: Nomor 89
  • Administrative Distance: 110
  • Multicast: 224.0.0.5 (All OSPF Routers), 224.0.0.6 (DR/BDR)
  • Metric: Cost (berbasis bandwidth) = 100 Mbps / bandwidth

🔄 Proses Pembentukan Adjacency OSPF ▼

OSPF membentuk adjacency (hubungan tetangga) melalui beberapa tahap:

Down → Init → 2-Way → ExStart → Exchange → Loading → Full
  • Down: Belum ada Hello diterima
  • Init: Hello diterima tapi RID kita belum ada di paket Hello tetangga
  • 2-Way: Melihat RID kita di Hello tetangga - DR/BDR dipilih di sini
  • ExStart: Negosiasi master/slave untuk pertukaran DBD
  • Exchange: Bertukar Database Description (ringkasan LSDB)
  • Loading: Meminta LSA yang belum dimiliki (LSR/LSU)
  • Full: Adjacency terbentuk penuh, LSDB sinkron
✅ Tips: State yang diinginkan adalah Full. State 2-Way pada non-DR/BDR adalah normal (hanya DR/BDR yang mencapai Full dengan semua router).

🏗️ Area OSPF & Jenis Router ▼

OSPF menggunakan konsep area untuk membagi jaringan menjadi bagian lebih kecil dan mengurangi overhead routing.

Area Backbone (Area 0): Area pusat yang harus ada. Semua area lain harus terhubung ke Area 0.

Jenis Router OSPF:

  • Internal Router: Semua interface dalam satu area
  • Backbone Router: Setidaknya satu interface di Area 0
  • ABR (Area Border Router): Interface di lebih dari satu area
  • ASBR (AS Boundary Router): Terhubung ke domain routing lain (redistribute)
👑 DR & BDR (Designated Router) ▼

Pada jaringan multi-access (Ethernet), semua router tidak perlu membentuk adjacency penuh dengan semua router lain (n*(n-1)/2 adjacency). DR dan BDR mengurangi jumlah adjacency.

Peran DR (Designated Router):

  • Semua router lain membentuk adjacency Full hanya dengan DR dan BDR
  • DR mendistribusikan LSA ke semua router
  • Mengurangi flooding LSA

Pemilihan DR/BDR:

  1. Priority tertinggi dipilih sebagai DR (default priority=1, range 0-255)
  2. Jika priority sama, Router ID tertinggi dipilih
  3. Priority=0 berarti router TIDAK mau menjadi DR/BDR

Router ID (RID) - Urutan Prioritas:

  1. RID yang dikonfigurasi manual (paling diprioritaskan)
  2. IP tertinggi dari loopback interface yang aktif
  3. IP tertinggi dari interface fisik yang aktif

4 Lab 19: Passive Interface
💡 Passive Interface: Interface yang di-passive tidak akan mengirim/menerima paket Hello OSPF. Network tetap diiklankan ke neighbor lain, tapi tidak ada adjacency yang terbentuk di interface tersebut. Digunakan untuk interface yang terhubung ke client (tidak perlu adjacency dengan PC).
1
/routing ospf interface add interface=ether1 passive=yes # ether1 adalah interface yang terhubung ke PC/client
2
/routing ospf interface print # ether1 harus muncul dengan passive=yes /routing ospf neighbor print # Tidak ada neighbor dari ether1

9 Lab 24: LSA Type 1 & Type 2
Penjelasan LSA Type 1 dan Type 2 ▼
  • LSA Type 1 (Router LSA): Dibuat oleh setiap router. Berisi informasi tentang interface dan link router tersebut. Hanya flood dalam satu area.
  • LSA Type 2 (Network LSA): Dibuat oleh DR. Berisi informasi tentang semua router yang terhubung ke jaringan multi-access. Hanya flood dalam satu area.
1
/routing ospf lsa print
2
/routing ospf lsa print where type=router # Muncul LSA dari setiap router (R1, R2, R3)
3
/routing ospf lsa print where type=network # Muncul LSA dari DR saja # Advertising-router = RID DR
10 LSA Types 3, 4, dan 5 (OSPF Multi-Area)

Selain LSA Type 1 (Router) dan Type 2 (Network) yang beroperasi dalam satu area, OSPF juga memiliki LSA yang digunakan untuk komunikasi antar area dan ke jaringan eksternal.

💡 Konteks: LSA Type 3, 4, dan 5 hanya relevan pada topologi OSPF multi-area (lebih dari satu area). Pada single-area OSPF seperti lab sebelumnya, hanya Type 1 dan Type 2 yang ada.

Type 1 Router LSA Setiap router Dalam satu area saja Mendeskripsikan link dan status interface router
Type 2 Network LSA DR (Designated Router) Dalam satu area saja Mendeskripsikan router yang terhubung ke segment multi-access
Type 3 Summary LSA ABRArea Border Router: router yang menghubungkan dua area OSPF berbeda Antar area (dari non-backbone ke backbone dan sebaliknya) Mengiklankan network dari satu area ke area lain (summary rute)
Type 4 ASBR Summary LSA ABR Antar area Memberitahu lokasi (IP) dari ASBRAS Boundary Router: router yang menghubungkan OSPF dengan routing protocol lain (misal RIP, BGP) agar router lain tahu cara mencapai rute eksternal
Type 5 AS External LSA ASBRAS Boundary Router: router yang menghubungkan OSPF dengan routing protocol lain Seluruh domain OSPF (semua area) Mengiklankan rute dari protokol lain (RIP, BGP, static) yang di-redistribusi ke OSPF

📝 Detail LSA Type 3 — Summary LSA ▼

Kapan dibuat? Saat sebuah ABR (router yang berada di dua area) perlu mengiklankan jaringan dari Area 1 ke Area 0, atau sebaliknya.

Analoginya: Seperti "buletin" yang dikirim oleh ABR ke area lain, berisi ringkasan prefix jaringan yang ada di area asalnya.

# Melihat LSA Type 3 di database OSPF MikroTik: /routing ospf lsa print where type=summary # atau pada v7: /routing/ospf/lsa print where type=summary-network
✅ LSA Type 3 hanya dikirim antar area — TIDAK disebarkan di dalam area yang sama. Ini yang menyebabkan router dalam satu area tidak melihat detail topologi area lain, hanya melihat summarized prefix-nya.
📝 Detail LSA Type 4 — ASBR Summary LSA ▼

Masalah yang dipecahkan: Saat ada ASBR di Area 1 yang meredistribusi rute eksternal (Type 5), router di Area 2 perlu tahu "di mana ASBR itu berada" agar bisa mencapai rute eksternal tersebut.

Siapa yang membuat? ABR yang menghubungkan area tempat ASBR berada dengan area lain.

Isi: Router ID dari ASBR + cost untuk mencapai ASBR tersebut.

Analogi: LSA Type 4 adalah "petunjuk jalan" menuju ASBR, sedangkan LSA Type 5 adalah "isi informasi rute eksternal" yang diiklankan ASBR.

📝 Detail LSA Type 5 — AS External LSA ▼

Kapan dibuat? Saat sebuah ASBR melakukan redistribusi — yaitu memasukkan rute dari protokol lain (RIP, BGP, static route) ke dalam domain OSPF.

Disebarkan ke: Seluruh domain OSPF (semua area), kecuali area Stub dan NSSA.

Metric Type:

  • Type 1 (E1): Cost dihitung dari ASBR + cost internal OSPF (lebih akurat)
  • Type 2 (E2): Cost hanya dari ASBR, tidak termasuk cost internal (default)
# Contoh redistribusi static route ke OSPF di MikroTik (membuat ASBR): /routing ospf instance set default redistribute-static=as-type-1 # atau type-2 (default) /routing ospf instance set default redistribute-static=as-type-2 # Melihat LSA Type 5 di database: /routing ospf lsa print where type=as-external
⚠️ Catatan penting: Area Stub tidak menerima LSA Type 5 (AS External). Sebagai gantinya, ABR mengirimkan sebuah default route (Type 3) ke area tersebut. Ini berguna untuk mengurangi ukuran LSDB di area yang tidak perlu tahu detail rute eksternal.
