1 Skenario: PT Jaringan Nusantara
🏢 Latar Belakang Perusahaan:

PT Jaringan Nusantara adalah perusahaan teknologi yang memiliki dua lokasi: Kantor Pusat di Banjarbaru dan Kantor Cabang di Pelaihari. Anda ditugaskan sebagai Network Engineer untuk merancang dan mengimplementasikan infrastruktur jaringan terintegrasi.

Persyaratan Jaringan:
  1. Kantor Pusat (AS 65010): Dual-WAN ke ISP1 dan ISP2 dengan failover dan load balancing
  2. VLAN untuk segmentasi: VLAN 10 (Staf), VLAN 20 (Lab Komputer)
  3. OSPF Area 0 sebagai IGP internal di kantor pusat
  4. eBGP antar kantor pusat dan cabang untuk konektivitas lintas site
  5. Kantor Cabang (AS 65020): LAN tunggal, koneksi eBGP ke pusat
⚠️ Studi Kasus ini adalah ujian praktikum komprehensif. Mahasiswa harus mengintegrasikan semua teknologi yang dipelajari di P1-P7: Static Routing, Fail Over, Routing Policy, OSPF, BGP, VLAN, dan Bridging.

3 Bagian A: Dual-WAN Failover & Load Balance
📚 Review: Materi P2 (Fail Over dan Load Balance)
💡 Konfigurasi: R-PUSAT memiliki dua koneksi ISP. ISP1 (ether1) adalah jalur utama, ISP2 (ether2) adalah jalur backup. Saat ISP1 down, traffic otomatis beralih ke ISP2.
1
# Sesuaikan dengan IP dari ISP (bisa DHCP atau static) /ip address add address=10.0.1.2/30 interface=ether1 # ISP1 /ip address add address=10.0.2.2/30 interface=ether2 # ISP2
2
# Jalur UTAMA via ISP1 (distance=1) /ip route add dst-address=0.0.0.0/0 gateway=10.0.1.1 \ distance=1 check-gateway=ping # Jalur BACKUP via ISP2 (distance=2) /ip route add dst-address=0.0.0.0/0 gateway=10.0.2.1 \ distance=2 check-gateway=ping
3
/ip firewall nat add chain=srcnat action=masquerade out-interface=ether1 /ip firewall nat add chain=srcnat action=masquerade out-interface=ether2
4
/ip route print # Route via ISP1 harus "A S" dengan distance=1 # Route via ISP2 harus "S" (tidak active) dengan distance=2
4 Bagian B: VLAN dan DHCP
📚 Review: Materi P7 (VLAN dan Bridging)
1
/interface bridge add name=br-pusat vlan-filtering=no /interface bridge port add bridge=br-pusat interface=ether3 /interface bridge vlan add bridge=br-pusat vlan-ids=10 \ tagged=br-pusat untagged=ether3 /interface bridge vlan add bridge=br-pusat vlan-ids=20 \ tagged=br-pusat untagged=ether3 /interface vlan add name=vlan10-staf vlan-id=10 interface=br-pusat /interface vlan add name=vlan20-lab vlan-id=20 interface=br-pusat /ip address add address=192.168.10.1/24 interface=vlan10-staf /ip address add address=192.168.20.1/24 interface=vlan20-lab /interface bridge set br-pusat vlan-filtering=yes
1
/ip pool add name=pool-staf ranges=192.168.10.100-192.168.10.200 /ip dhcp-server network add \ address=192.168.10.0/24 \ gateway=192.168.10.1 \ dns-server=8.8.8.8,8.8.4.4 /ip dhcp-server add name=dhcp-staf interface=vlan10-staf \ address-pool=pool-staf disabled=no
2
/ip pool add name=pool-lab ranges=192.168.20.100-192.168.20.200 /ip dhcp-server network add \ address=192.168.20.0/24 \ gateway=192.168.20.1 \ dns-server=8.8.8.8,8.8.4.4 /ip dhcp-server add name=dhcp-lab interface=vlan20-lab \ address-pool=pool-lab disabled=no

5 Bagian C: OSPF IGP Internal
📚 Review: Materi P5 (OSPF MikroTik)
💡 OSPF digunakan sebagai IGP (Internal Gateway Protocol) di dalam kantor pusat. Semua network internal (VLAN 10, VLAN 20, link ke cabang) diiklankan melalui OSPFOpen Shortest Path First: protokol routing dinamis yang mencari jalur terpendek Area 0.
1
/routing ospf instance add name=default router-id=10.10.10.1
2
/routing ospf network add network=192.168.10.0/24 area=backbone /routing ospf network add network=192.168.20.0/24 area=backbone /routing ospf network add network=10.10.45.0/30 area=backbone
3
/routing ospf interface add interface=vlan10-staf passive=yes /routing ospf interface add interface=vlan20-lab passive=yes
4
/routing ospf neighbor print /ip route print where routing-protocol="ospf"
6 Bagian D: eBGP Antar Site
📚 Review: Materi P6 (BGP MikroTik)
💡 eBGP digunakan untuk menghubungkan Kantor Pusat (AS 65010) dan Kantor Cabang (AS 65020). Setiap site mengiklankan network lokalnya melalui BGPBorder Gateway Protocol: protokol routing antar jaringan besar/ISP.
1
/ip address add address=10.10.45.1/30 interface=ether4
2
/routing bgp template add name=to-cabang as=65010 \ router-id=10.10.45.1 /routing bgp connection add name=ke-cabang \ template=to-cabang \ remote.address=10.10.45.2 \ remote.as=65020 \ local.address=10.10.45.1
3
/routing bgp network add network=192.168.10.0/24 /routing bgp network add network=192.168.20.0/24
1
/ip address add address=10.10.45.2/30 interface=ether1 /ip address add address=192.168.30.1/24 interface=ether2
2
/routing bgp template add name=to-pusat as=65020 \ router-id=10.10.45.2 /routing bgp connection add name=ke-pusat \ template=to-pusat \ remote.address=10.10.45.1 \ remote.as=65010 \ local.address=10.10.45.2 /routing bgp network add network=192.168.30.0/24
3
/ip route add dst-address=0.0.0.0/0 gateway=10.10.45.1 /ip firewall nat add chain=srcnat action=masquerade out-interface=ether1
1
# Di R-PUSAT dan R-CABANG: /routing bgp session print # State harus: established
2
# Di R-PUSAT: /ip route print where routing-protocol="bgp" # Harus muncul: A b 192.168.30.0/24 10.10.45.2 20 # Di R-CABANG: /ip route print where routing-protocol="bgp" # Harus muncul: A b 192.168.10.0/24 10.10.45.1 20 # A b 192.168.20.0/24 10.10.45.1 20
7 Bagian E: Verifikasi Menyeluruh
✅ Checklist Verifikasi: Centang setiap item setelah berhasil diverifikasi. Semua item harus terpenuhi untuk jaringan yang berfungsi penuh.

1. Test Konektivitas Internet

1
ping 8.8.8.8 # dari PC Staf # Expected: Reply from 8.8.8.8
2
ping 8.8.8.8 # dari PC Lab # Expected: Reply from 8.8.8.8

2. Test Inter-VLAN

1
ping 192.168.20.1 # dari PC Staf tracert 192.168.20.1 # harus lewat 192.168.10.1

3. Test BGP dan Konektivitas Lintas Site

1
/routing bgp session print # Harus: established
2
ping 192.168.30.1 # dari PC Staf ke gateway cabang tracert 192.168.30.1 # harus lewat 10.10.45.1 → 10.10.45.2
3
ping 192.168.10.1 # dari PC Cabang ke gateway VLAN Staf

4. Test Failover WAN

1
# Matikan ether1 (ISP1) di R-PUSAT /interface disable ether1 # Tunggu ~10 detik # Cek routing table /ip route print # Route via ISP2 harus menjadi Active # Test konektivitas masih berjalan: ping 8.8.8.8 # Harus masih reply (via ISP2)
2
/interface enable ether1 # Tunggu check-gateway berhasil /ip route print # Route via ISP1 kembali Active (distance=1 diprioritaskan)
