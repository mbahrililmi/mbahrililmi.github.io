1 Skenario: PT Jaringan Nusantara
🏢 Latar Belakang Perusahaan:

PT Jaringan Nusantara adalah perusahaan teknologi yang memiliki dua lokasi: Kantor Pusat di Banjarbaru dan Kantor Cabang di Pelaihari. Anda ditugaskan sebagai Network Engineer untuk merancang dan mengimplementasikan infrastruktur jaringan terintegrasi.

Persyaratan Jaringan:
  1. Kantor Pusat (AS 65010): Dual-WAN ke ISP1 dan ISP2 dengan failover dan load balancing
  2. VLAN untuk segmentasi: VLAN 10 (Staf), VLAN 20 (Lab Komputer)
  3. OSPF Area 0 sebagai IGP internal di kantor pusat
  4. eBGP antar kantor pusat dan cabang untuk konektivitas lintas site
  5. Kantor Cabang (AS 65020): LAN tunggal, koneksi eBGP ke pusat
⚠️ Studi Kasus ini adalah ujian praktikum komprehensif. Mahasiswa harus mengintegrasikan semua teknologi yang dipelajari di P1-P7: Static Routing, Fail Over, Routing Policy, OSPF, BGP, VLAN, dan Bridging.

3 Bagian A: Dual-WAN Failover & Load Balance
📚 Review: Materi P2 (Fail Over dan Load Balance)
💡 Konfigurasi: R-PUSAT memiliki dua koneksi ISP. ISP1 (ether1) adalah jalur utama, ISP2 (ether2) adalah jalur backup. Saat ISP1 down, traffic otomatis beralih ke ISP2.
1
# Sesuaikan dengan IP dari ISP (bisa DHCP atau static) /ip address add address=10.0.1.2/30 interface=ether1 # ISP1 /ip address add address=10.0.2.2/30 interface=ether2 # ISP2
2
# Jalur UTAMA via ISP1 (distance=1) /ip route add dst-address=0.0.0.0/0 gateway=10.0.1.1 \ distance=1 check-gateway=ping # Jalur BACKUP via ISP2 (distance=2) /ip route add dst-address=0.0.0.0/0 gateway=10.0.2.1 \ distance=2 check-gateway=ping
3
/ip firewall nat add chain=srcnat action=masquerade out-interface=ether1 /ip firewall nat add chain=srcnat action=masquerade out-interface=ether2
4
/ip route print # Route via ISP1 harus "A S" dengan distance=1 # Route via ISP2 harus "S" (tidak active) dengan distance=2
