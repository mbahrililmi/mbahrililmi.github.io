📖 Konsep Routing (Layer 3 OSI) ▼

RoutingProses memilih jalur terbaik untuk mengirim paket data adalah proses penentuan jalur terbaik untuk meneruskan paket data dari sumber ke tujuan melalui jaringan. Routing bekerja di Layer 3 (Network Layer) model OSI menggunakan IP Address.

Terdapat dua jenis routing utama:

  • Static Routing: Rute dikonfigurasi manual oleh administrator. Cocok untuk jaringan kecil dan stabil.
  • Dynamic Routing: Rute dipelajari otomatis menggunakan protokol seperti OSPFOpen Shortest Path First: protokol routing dinamis yang mencari jalur terpendek atau BGPBorder Gateway Protocol: protokol routing antar jaringan besar/ISP.
💡 Catatan: Static routing tidak memerlukan overhead pemrosesan CPU seperti dynamic routing, namun membutuhkan konfigurasi manual setiap ada perubahan topologi.
📋 Routing Table dan Flag Routing ▼

Routing table adalah database yang menyimpan informasi jalur ke jaringan-jaringan tujuan. Setiap entri routing table memiliki flag yang menunjukkan status dan jenis route:

FlagKeteranganContoh
AActive - route sedang aktif digunakanA S 192.168.1.0/24
DDynamic - route dari protokol dinamisA D 10.0.0.0/8
CConnected - interface langsung terhubungA C 192.168.0.0/24
SStatic - route dikonfigurasi manualA S 172.16.0.0/16
oOSPF - route dari protokol OSPFA o 10.1.0.0/24
bBGP - route dari protokol BGPA b 203.0.113.0/24
⚠️ Penting: Route hanya digunakan jika memiliki flag A (Active). Route tanpa flag A berarti tidak aktif dan tidak digunakan untuk forwarding paket.
🔢 Administrative Distance ▼

Administrative Distance (AD) adalah nilai kepercayaan suatu sumber routing. Semakin kecil nilai AD, semakin dipercaya dan diprioritaskan route tersebut.

Sumber RouteAdministrative DistancePrioritas
Connected0Tertinggi
Static1Sangat Tinggi
eBGP20Tinggi
OSPFOpen Shortest Path First: protokol routing dinamis yang mencari jalur terpendek110Sedang
RIP120Rendah
iBGP200Sangat Rendah
✅ Tips: Jika ada dua route ke tujuan yang sama dengan AD berbeda, router akan memilih route dengan AD lebih kecil. Jika AD sama, lihat metric (prefix length yang lebih panjang = lebih spesifik = diprioritaskan).
🗂️ Route Summarization dan Default Route ▼

Route Summarization (agregasi route) adalah teknik menggabungkan beberapa route spesifik menjadi satu route yang lebih umum untuk menyederhanakan routing table.

Contoh Summarization:

  • 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24
  • Dapat diringkas menjadi: 192.168.0.0/22
  • Alasan: 4 network (/24) = 2² → pinjam 2 bit dari host bit → /22

Default Route (Gateway of Last Resort):

Default route menggunakan prefix 0.0.0.0/0 dan cocok dengan SEMUA tujuan yang tidak ada di routing table. Digunakan sebagai "jalan terakhir" ketika tidak ada route spesifik yang cocok.

Point-to-Point Addressing (/32):

Prefix /32 menunjuk pada satu host spesifik. Digunakan untuk menentukan gateway yang tidak langsung terhubung melalui subnetPembagian jaringan menjadi bagian-bagian lebih kecil yang sama.

💡 Rumus Summarization: Jumlah network yang diringkas harus merupakan pangkat 2 (2, 4, 8, 16...). Network pertama harus merupakan kelipatan dari jumlah tersebut.

💡 Skenario: PC1 (192.168.1.0/24) terhubung ke R1, PC2 (192.168.2.0/24) terhubung ke R2. R1 dan R2 dihubungkan via link 10.10.10.0/30. Konfigurasi static route bidirectional agar PC1 dan PC2 dapat saling berkomunikasi.

Konfigurasi R1:

1
/ip address add address=10.10.10.1/30 interface=ether1
2
/ip address add address=192.168.1.1/24 interface=ether2
3
/ip route add dst-address=192.168.2.0/24 gateway=10.10.10.2
4
/ip route print

Konfigurasi R2:

1
/ip address add address=10.10.10.2/30 interface=ether1
2
/ip address add address=192.168.2.1/24 interface=ether2
3
/ip route add dst-address=192.168.1.0/24 gateway=10.10.10.1

Output Routing Table (R1) — yang diharapkan:

Flags: X - disabled, A - active, D - dynamic, C - connect, S - static, r - rip, b - bgp, o - ospf # DST-ADDRESS PREF-SRC GATEWAY DISTANCE 0 A S 192.168.2.0/24 10.10.10.2 1 1 A C 192.168.1.0/24 192.168.1.1 ether2 0 2 A C 10.10.10.0/30 10.10.10.1 ether1 0
✅ Tips Verifikasi: Pastikan flag A S muncul pada static route yang ditambahkan. "A" berarti Active, "S" berarti Static. Jika hanya "S" tanpa "A", periksa apakah gateway dapat dijangkau.

6 Lab 4: Route Summarization
💡 Skenario: R1 memiliki 4 network LAN: 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24. Alih-alih memasukkan 4 route di R2, gunakan summarization menjadi 1 route 192.168.0.0/22.
Perhitungan Summarization ▼

Langkah perhitungan:

  1. Network: 192.168.0.0, 192.168.1.0, 192.168.2.0, 192.168.3.0
  2. Jumlah network = 4 = 2² → perlu meminjam 2 bit
  3. Prefix asli = /24, pinjam 2 bit ke kiri → /22
  4. Summary address = 192.168.0.0/22 (mencakup .0.0 s/d .3.255)
Verifikasi: 192.168.0.0/22 mencakup: 192.168.0.0 - 192.168.0.255 192.168.1.0 - 192.168.1.255 192.168.2.0 - 192.168.2.255 192.168.3.0 - 192.168.3.255
1
# Di R2: satu summary route menggantikan 4 route spesifik /ip route add dst-address=192.168.0.0/22 gateway=10.10.10.1
2
/ip route print # Seharusnya hanya 1 route A S untuk 192.168.0.0/22

ping 192.168.0.1 ping 192.168.1.1 ping 192.168.2.1 ping 192.168.3.1
7 Lab 5: Default Route (0.0.0.0/0)
💡 Default Route dengan prefix 0.0.0.0/0 akan cocok dengan semua tujuan yang tidak ada entri spesifiknya di routing table. Ini adalah "jalur terakhir" — gateway of last resort.
1
# Default route - cocok dengan semua tujuan /ip route add dst-address=0.0.0.0/0 gateway=10.10.10.2
2
/ip route print # Lihat baris dengan DST-ADDRESS = 0.0.0.0/0 dan flag A S
3
ping 8.8.8.8 # Paket akan diteruskan via default route jika tidak ada route spesifik
8 Lab 6: Point-to-Point Addressing (/32)
R1 (10.10.10.10/32) ──── Link Point-to-Point ──── R2 Loopback/host: 10.10.10.10 Network tujuan: 20.20.20.20/32

Point-to-point addressing menggunakan prefix /32 yang menunjuk pada satu host tunggal. Berguna untuk routing ke gateway yang tidak berada di subnetPembagian jaringan menjadi bagian-bagian lebih kecil yang sama.

1
# Route ke host 20.20.20.20 menggunakan /32 /ip route add dst-address=20.20.20.20/32 gateway=10.10.10.10 # Atau: route ke network lain via gateway yang ditentukan dengan /32 /ip route add dst-address=192.168.5.0/24 gateway=10.10.10.10
2
/ip route print where dst-address="20.20.20.20/32"
9 Lab 7: Prioritas Routing

Router menentukan route terbaik berdasarkan dua kriteria utama (dari yang paling diprioritaskan):

  1. Prefix Length (Longest Match): Prefix yang lebih panjang/spesifik selalu menang, terlepas dari AD
  2. Administrative Distance: Jika prefix sama, AD lebih kecil diprioritaskan
RoutePrefixADDipilih?
Static ke 192.168.1.0/24/241Ya (lebih spesifik)
Static ke 192.168.0.0/22/221Tidak (kurang spesifik)
Default 0.0.0.0/0/01Hanya jika tidak ada yang cocok
OSPF ke 10.0.0.0/8/8110Bergantung pada prefix tujuan
⚠️ Penting: Longest Prefix Match SELALU menang atas Administrative Distance. Route /32 akan selalu dipilih daripada /0 meskipun AD-nya lebih tinggi.
1
# Route spesifik (lebih diprioritaskan) /ip route add dst-address=192.168.1.10/32 gateway=10.10.10.2 # Route umum /ip route add dst-address=192.168.1.0/24 gateway=10.10.10.3 # Default route (paling rendah prioritas) /ip route add dst-address=0.0.0.0/0 gateway=10.10.10.4
2
# Cek route yang akan digunakan untuk tujuan tertentu /ip route check 192.168.1.10
3
/tool traceroute 192.168.1.10
