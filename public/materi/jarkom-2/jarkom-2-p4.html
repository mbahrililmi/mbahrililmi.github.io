Firewall, NAT dan Fasttrack

Konfigurasi Firewall Filter, Address-List, Source NAT, Destination NAT, dan FastTrack di MikroTik

1 Tujuan Pembelajaran

🎯 Setelah menyelesaikan praktikum ini, mahasiswa mampu:

  1. Menjelaskan konsep dan prinsip dasar FirewallSistem keamanan jaringan yang memantau lalu lintas
  2. Mengkonfigurasi Firewall Filter pada MikroTik RouterOS
  3. Menggunakan Address-ListFitur pengelompokan IP ke dalam daftar bernama untuk manajemen IP
  4. Memahami dan mengkonfigurasi Source NATMengubah IP sumber agar LAN bisa akses internet (Masquerade)
  5. Memahami dan mengkonfigurasi Destination NATMengubah IP tujuan untuk expose server LAN ke internet (Port Forwarding)
  6. Mengaktifkan FastTrackFitur MikroTik untuk mempercepat pemrosesan paket di kernel untuk optimasi throughput

2 Teori Firewall

🛡️ Prinsip Dasar Firewall ▼

Firewall adalah sistem keamanan yang memantau dan mengontrol lalu lintas jaringan masuk dan keluar berdasarkan aturan keamanan yang telah ditentukan.

Terdapat 2 prinsip dasar Firewall:

PrinsipDeskripsiRekomendasi
Default DENYBlokir semua, izinkan yang spesifik✅ DIREKOMENDASIKAN
Default ALLOWIzinkan semua, blokir yang spesifik❌ Kurang aman
💡
Gunakan prinsip Default DENY untuk keamanan maksimal. Lebih baik memblokir semua kemudian mengizinkan yang diperlukan.
🔗 Chain INPUT / FORWARD / OUTPUT ▼

MikroTik Firewall Filter memiliki 3 chain utama:

ChainDeskripsiContoh Penggunaan
INPUTPaket ditujukan ke router sendiriAkses Winbox, SSH, PING ke router
OUTPUTPaket berasal dari router sendiriRouter kirim update, DNS query
FORWARDPaket diteruskan melalui routerPC LAN akses internet, antar LAN

Actions yang tersedia:

  • accept — Izinkan paket lewat
  • drop — Buang paket diam-diam (tanpa notifikasi ke pengirim)
  • reject — Buang paket dan kirim notifikasi ke pengirim
  • log — Catat informasi paket ke log
  • jump — Lompat ke chain lain
📡 Connection State ▼

MikroTik melacak kondisi setiap koneksi jaringan (stateful firewall):

StateDeskripsi
establishedKoneksi yang sudah ada dan aktif
relatedKoneksi terkait dengan koneksi yang sudah ada (mis. FTP data)
newPaket pertama dari koneksi baru
invalidPaket tidak sesuai state koneksi mana pun (harus di-drop)
ℹ️
Aturan pertama yang direkomendasikan: accept established,related dan drop invalid untuk optimasi pemrosesan firewall.
📋 Address-List ▼

Address-List adalah fitur MikroTik untuk mengelompokkan satu atau lebih alamat IP ke dalam sebuah daftar bernama. Berguna untuk manajemen firewall yang lebih efisien.

Kegunaan utama:

  • Blacklist: Daftar IP yang diblokir aksesnya
  • Whitelist: Daftar IP yang selalu diizinkan
  • Manajemen kelompok IP (mis. IP server, IP admin)
  • Membuat aturan firewall lebih simpel dan mudah dikelola
🔄 NAT (Network Address Translation) ▼

NATNetwork Address Translation: menerjemahkan alamat IP dalam paket memungkinkan beberapa perangkat berbagi satu alamat IP publik.

Jenis NATChainActionFungsi
Source NAT (SNAT / Masquerade)srcnatmasqueradeUbah IP SUMBER agar LAN bisa akses internet melalui satu IP publik
Destination NAT (DNAT / Port Forwarding)dstnatdst-natUbah IP TUJUAN untuk meneruskan koneksi ke server di LAN
⚡ FastTrack ▼

FastTrack adalah fitur MikroTik yang memproses paket secara langsung di level kernel tanpa melalui full firewall processing. Hal ini dapat meningkatkan throughput secara signifikan.

FastTrack mem-bypass:

  • Mangle rules
  • Queue (Simple Queue dan Queue Tree)
  • Sebagian besar pemrosesan di layer atas kernel
⚠️
PERINGATAN: FastTrack mem-bypass proses connection tracking dan sebagian firewall! Gunakan dengan hati-hati. Jika menggunakan QoS/Mangle, FastTrack bisa mengganggu fungsionalitasnya.

3 Praktikum A: Firewall Filter Dasar

ℹ️
Konfigurasi firewall filter dasar menggunakan prinsip Default Deny pada MikroTik RouterOS.

Topologi Jaringan:

[Internet] --- ether1 [MikroTik] ether2 --- [LAN 192.168.1.0/24]
1

Login ke MikroTik via Winbox

Buka aplikasi Winbox, masukkan IP atau MAC address MikroTik, login dengan username dan password.

2

Allow established/related INPUT:

RouterOS CLI
/ip firewall filter add chain=input connection-state=established,related action=accept comment="Allow established/related INPUT"
3

Drop invalid INPUT:

RouterOS CLI
/ip firewall filter add chain=input connection-state=invalid action=drop comment="Drop invalid INPUT"
4

Allow ICMP dari LAN:

RouterOS CLI
/ip firewall filter add chain=input protocol=icmp src-address=192.168.1.0/24 action=accept comment="Allow ICMP from LAN"
5

Allow Winbox dari LAN (port TCP 8291):

RouterOS CLI
/ip firewall filter add chain=input protocol=tcp dst-port=8291 src-address=192.168.1.0/24 action=accept comment="Allow Winbox"
6

Drop semua INPUT dari internet (Default Deny):

RouterOS CLI
/ip firewall filter add chain=input in-interface=ether1 action=drop comment="Drop all from internet"
7

Allow FORWARD dari LAN ke internet:

RouterOS CLI
/ip firewall filter add chain=forward in-interface=ether2 out-interface=ether1 action=accept comment="Allow LAN to internet"
⚠️
Urutan Sangat Penting! Aturan firewall diproses dari atas ke bawah. Pastikan aturan accept established/related berada di paling atas untuk performa optimal.

4 Praktikum B: Address-List

ℹ️
Menggunakan Address-List untuk memblokir IP tertentu secara mudah dan terorganisir dengan konsep blacklist.
1

Buat Address-List blacklist dan tambahkan IP:

RouterOS CLI
/ip firewall address-list add list=blacklist address=192.168.1.50
2

Buat aturan firewall untuk drop IP dalam blacklist:

RouterOS CLI
/ip firewall filter add chain=forward src-address-list=blacklist action=drop comment="Block blacklisted IPs"
3

Uji konfigurasi:

Dari komputer dengan IP yang ada di blacklist (192.168.1.50), coba akses internet. Akses harus ditolak.

💡
Anda dapat menambahkan beberapa IP ke dalam satu Address-List dengan menjalankan perintah /ip firewall address-list add berulang kali menggunakan nama list yang sama.

5 Praktikum C: Source NAT (Masquerade)

ℹ️
Source NAT (Masquerade) memungkinkan semua perangkat di LAN mengakses internet menggunakan satu IP publik milik router.
1

Tambahkan aturan masquerade pada interface WAN (ether1):

RouterOS CLI
/ip firewall nat add chain=srcnat out-interface=ether1 action=masquerade comment="Masquerade LAN ke Internet"
2

Verifikasi — ping ke internet dari PC di LAN:

Windows CMD / Linux Terminal
ping 8.8.8.8

6 Praktikum D: Destination NAT (Port Forwarding)

ℹ️
Destination NAT (Port Forwarding) memungkinkan server di jaringan LAN dapat diakses dari internet melalui IP publik router.

Skenario:

Web server berada di IP 192.168.1.100 port 80. Akses dari internet melalui port 8080 pada IP publik router.

[Internet] --port 8080--> [MikroTik IP Publik:8080] | v (dst-nat) | [192.168.1.100:80] (Web Server LAN)
1

Tambahkan aturan DNAT (Port Forwarding):

RouterOS CLI
/ip firewall nat add chain=dstnat protocol=tcp dst-port=8080 action=dst-nat to-addresses=192.168.1.100 to-ports=80 comment="Port Forward 8080 ke Web Server"
2

Verifikasi akses dari internet:

Browser URL
http://[IP-publik-router]:8080
⚠️
Pastikan juga sudah mengizinkan port 8080 di aturan firewall filter INPUT atau FORWARD agar koneksi tidak terblokir oleh firewall.

7 Praktikum E: FastTrack

ℹ️
FastTrack meningkatkan throughput dengan memproses paket established/related secara langsung di kernel, melewati proses firewall penuh.
1

Tambahkan aturan fasttrack-connection untuk koneksi established/related:

RouterOS CLI
/ip firewall filter add chain=forward connection-state=established,related action=fasttrack-connection comment="FastTrack established/related"
2

Tambahkan aturan accept setelah FastTrack:

RouterOS CLI
/ip firewall filter add chain=forward connection-state=established,related action=accept comment="Accept established/related"
3

Verifikasi dengan Tools → Torch:

Di Winbox, buka Tools → Torch, pilih interface ether1. Amati trafik dan throughput yang meningkat setelah FastTrack diaktifkan.

🚨
PENTING — Urutan Aturan: Pastikan aturan FastTrack berada DI ATAS aturan drop. Urutan aturan firewall sangat penting karena diproses dari atas ke bawah. FastTrack mem-bypass QoS dan Mangle — tidak cocok digunakan bersama konfigurasi bandwidth management!

8 Rubrik Penilaian

1
Firewall Filter Dasar — Filter INPUT/FORWARD terkonfigurasi dengan benar sesuai prinsip Default Deny
2
Address-List — IP yang di-blacklist berhasil diblokir dan terverifikasi pengujiannya
3
Source NAT Masquerade — LAN dapat mengakses internet melalui satu IP publik router
4
Destination NAT Port Forwarding — Server internal dapat dijangkau dari internet melalui IP publik router
5
FastTrack — FastTrack aktif dan throughput meningkat (diverifikasi dengan Torch)

9 Tugas Laporan

  • 1Implementasikan Firewall Filter dengan kebijakan Default Deny. Dokumentasikan setiap aturan beserta alasannya!
  • 2Buat Address-List untuk memblokir minimal 5 alamat IP. Uji dan dokumentasikan hasilnya!
  • 3Konfigurasi Source NAT (Masquerade) agar PC di LAN dapat mengakses internet. Sertakan bukti ping ke 8.8.8.8!
  • 4Konfigurasi Port Forwarding untuk meneruskan koneksi port 80 dari internet ke web server di jaringan lokal. Uji dan dokumentasikan!
  • 5Aktifkan FastTrack dan jelaskan dampaknya terhadap kinerja jaringan berdasarkan hasil pengujian!
