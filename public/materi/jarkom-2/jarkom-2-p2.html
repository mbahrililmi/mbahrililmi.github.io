2 Teori Routing

Routing adalah proses memilih jalur terbaik untuk mengirimkan paket data dari sumber ke tujuan melalui jaringan yang saling terhubung. Router menggunakan routing table untuk menentukan ke mana paket harus dikirimkan berdasarkan IP tujuan.

💡 Router membuat keputusan routingProses memilih jalur terbaik untuk mengirim paket data berdasarkan Longest Prefix Match — rute dengan prefix terpanjang (spesifik) yang cocok dengan IP tujuan akan dipilih.
📌 Static Routing (Routing Statis) ▼

Static routing dikonfigurasi secara manual oleh administrator. Router tidak belajar rute secara otomatis. Cocok untuk jaringan kecil atau koneksi ke satu arah (stub network).

Kelebihan: Aman, tidak ada overhead protokol, mudah diprediksi
Kekurangan: Tidak scalable, tidak ada failover otomatis, administrasi manual

🔄 RIP (Routing Information Protocol) ▼

RIP adalah protokol Distance Vector yang menggunakan algoritma Bellman-Ford. Metrik RIP adalah hop count (jumlah router yang dilewati).

  • Hop count maksimum: 15 (hop ke-16 dianggap tidak terjangkau)
  • Update interval: 30 detik (broadcast/multicast tabel routing)
  • Konvergensi: Lambat (~beberapa menit untuk jaringan besar)
  • RIPv1: Classful, broadcast — RIPv2: Classless, multicast 224.0.0.9, mendukung VLSM
⚠️ RIP tidak cocok untuk jaringan besar karena batas 15 hop dan konvergensi yang lambat. Gunakan OSPF atau BGP untuk jaringan besar.
⚡ OSPF (Open Shortest Path First) ▼

OSPFOpen Shortest Path First: protokol routing dinamis yang mencari jalur terpendek adalah protokol Link State yang menggunakan algoritma Dijkstra (SPF) untuk menghitung jalur terpendek.

  • Area 0 (Backbone): Area pusat yang wajib ada, semua area lain harus terhubung ke Area 0
  • DR/BDR: Designated Router / Backup DR untuk mengurangi traffic LSA pada segmen multi-access
  • LSA (Link State Advertisement): Pesan yang berisi informasi topologi jaringan
  • LSDB: Database state link, dimiliki setiap router OSPF di area yang sama
  • Metrik: Cost (berdasarkan bandwidth: Cost = 10^8 / bandwidth)
  • Konvergensi: Cepat, cocok untuk jaringan enterprise besar
🌐 BGP (Border Gateway Protocol) ▼

BGPBorder Gateway Protocol: protokol routing antar jaringan besar/ISP adalah protokol Path Vector yang digunakan untuk routingProses memilih jalur terbaik untuk mengirim paket data antar Autonomous System (AS). BGP adalah protokol routing internet global.

  • Port: TCP 179
  • iBGP: BGP di dalam satu AS
  • eBGP: BGP antar AS yang berbeda
  • AS Number: Identifier unik setiap jaringan/organisasi (1-65535 publik, 64512-65535 privat)
  • Atribut: AS-PATH, NEXT-HOP, LOCAL-PREF, MED — digunakan untuk pemilihan rute
💡 BGP mengelola lebih dari 900.000 prefix di internet global (per 2024). Setiap ISP besar menggunakan BGP untuk pertukaran rute dengan ISP lain.
