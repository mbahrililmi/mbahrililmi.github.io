QoS (Quality of Service)

Manajemen bandwidth dengan Simple Queue, PCQ, dan Burst di MikroTik RouterOS

Kode: EIK212305
Semester: 2
Dosen: M. Bahril Ilmi, S.Kom., M.Kom.

1 Tujuan Pembelajaran

🎯 Setelah menyelesaikan praktikum ini, mahasiswa mampu:

  1. Menjelaskan konsep QoSQuality of Service: teknologi mengatur prioritas lalu lintas jaringan (Quality of Service)
  2. Memahami perbedaan Simple QueueMetode limit bandwidth per IP, paling sederhana di MikroTik dan Queue Tree
  3. Mengkonfigurasi Simple Queue untuk membatasi bandwidth per pengguna
  4. Mengkonfigurasi PCQPer Connection Queue: membagi bandwidth secara adil dan dinamis (Per Connection Queue) untuk pembagian bandwidth adil
  5. Melakukan pengujian dan verifikasi konfigurasi QoS

2 Teori QoS

📡 Konsep QoS (Quality of Service) ▼

QoS (Quality of Service) adalah seperangkat teknologi yang mengatur dan memprioritaskan lalu lintas jaringan agar bandwidth digunakan secara optimal dan adil. Tanpa QoS, satu pengguna berpotensi menghabiskan seluruh bandwidth jaringan.

Parameter QoS Penting:

Parameter Deskripsi Nilai Ideal (VoIP)
Bandwidth Kapasitas maksimum jalur jaringan (Mbps/Kbps) Sesuai kebutuhan
Throughput Bandwidth aktual yang berhasil digunakan Mendekati bandwidth
Latency Waktu yang dibutuhkan paket mencapai tujuan (ms) < 150 ms
Jitter Variasi keterlambatan antar paket (ms) < 30 ms
Packet Loss Persentase paket yang hilang dalam transmisi < 1%
⚙️ Mekanisme QoS di MikroTik ▼
Mekanisme Deskripsi Kelebihan
Simple Queue Limit bandwidth per IP/subnet, paling sederhana Mudah dikonfigurasi
Queue Tree Hierarki queue parent-child, berbasis Mangle Sangat fleksibel
PCQ (Per Connection Queue) Membagi bandwidth adil antar pengguna aktif secara dinamis Otomatis, adil, tidak perlu setting per IP

Format Max-Limit pada Simple Queue: [upload]/[download]
Contoh: 1M/2M artinya upload max 1 Mbps, download max 2 Mbps.

⚖️ PCQ (Per Connection Queue) ▼

PCQ membagi total bandwidth secara merata kepada semua pengguna yang aktif secara dinamis:

  • 1 pengguna aktif → mendapat semua bandwidth
  • 2 pengguna aktif → masing-masing mendapat 1/2 bandwidth
  • 5 pengguna aktif → masing-masing mendapat 1/5 bandwidth

PCQ Classifier:

  • dst-address → untuk download (trafik menuju PC)
  • src-address → untuk upload (trafik dari PC)
💡
PCQ sangat cocok untuk hotspot atau jaringan publik di mana jumlah pengguna tidak dapat diprediksi dan semua pengguna berhak mendapat porsi bandwidth yang adil.
🚀 Burst ▼

Burst adalah fitur yang mengizinkan pengguna untuk melebihi batas bandwidth normal untuk waktu singkat. Sangat berguna untuk membuat browsing terasa cepat walaupun limit bandwidth rendah.

Parameter Deskripsi
burst-limit Batas kecepatan maksimum saat burst aktif
burst-threshold Rata-rata kecepatan di bawah nilai ini agar burst aktif
burst-time Periode waktu untuk menghitung rata-rata kecepatan (detik)

3 Praktikum A: Simple Queue (Limit per User)

ℹ️
Membatasi bandwidth per pengguna menggunakan Simple Queue, metode paling sederhana di MikroTik.

Topologi Jaringan:

[Internet 10Mbps/10Mbps] | ether1 (IP Publik) [MikroTik RouterBoard] ether2 (192.168.1.1/24) | [LAN Switch] | +--------+--------+--------+ | | | | PC1 PC2 PC3 PC4 .10 .20 .30 .40
1

Login ke MikroTik via Winbox

Buka Winbox, masukkan IP/MAC MikroTik, login dengan kredensial admin.

2

Buka Queues → Simple Queues di Winbox

Navigasi ke menu Queues → tab Simple Queues untuk melihat daftar queue yang ada.

3

Buat queue untuk PC1 (download 2Mbps / upload 1Mbps):

RouterOS CLI 
/queue simple add name=PC1-Limit target=192.168.1.10 max-limit=1M/2M comment="Limit PC1"
4

Buat queue untuk PC2 dan PC3:

RouterOS CLI 
/queue simple add name=PC2-Limit target=192.168.1.20 max-limit=512k/1M
/queue simple add name=PC3-Limit target=192.168.1.30 max-limit=2M/4M
5

Verifikasi daftar queue di Winbox:

Buka Queues → Simple Queues. Pastikan semua queue muncul dengan status aktif (centang hijau).

6

Uji dengan speedtest.net:

Dari masing-masing PC, akses speedtest.net. Hasil kecepatan tidak boleh melebihi limit yang telah dikonfigurasi.

4 Praktikum B: PCQ (Bandwidth Sharing Adil)

ℹ️
PCQ (Per Connection Queue) membagi bandwidth secara otomatis dan adil kepada semua pengguna yang aktif, tanpa harus mengatur limit per IP.
1

Buat Queue Type untuk PCQ Download dan Upload:

RouterOS CLI 
/queue type add name=PCQ-Download kind=pcq pcq-rate=0 pcq-classifier=dst-address
/queue type add name=PCQ-Upload kind=pcq pcq-rate=0 pcq-classifier=src-address
2

Buat Simple Queue menggunakan PCQ untuk seluruh subnet LAN:

RouterOS CLI 
/queue simple add name=Internet-Sharing target=192.168.1.0/24 max-limit=10M/10M queue=PCQ-Upload/PCQ-Download comment="PCQ - Fair bandwidth sharing"
3

Verifikasi pembagian bandwidth yang adil:

  • 1 pengguna aktif download → gunakan hingga 10 Mbps
  • 2 pengguna aktif download → masing-masing hingga 5 Mbps
  • 4 pengguna aktif download → masing-masing hingga 2,5 Mbps

Uji dengan mendownload file besar bersamaan dari beberapa PC.

💡
Nilai pcq-rate=0 berarti tidak ada batas per koneksi — router akan membagi total bandwidth secara otomatis. Jika ingin membatasi per koneksi, isi nilai pcq-rate (mis. pcq-rate=2M).

5 Praktikum C: Burst

ℹ️
Burst memungkinkan pengguna melebihi batas bandwidth sesaat saat baru memulai unduhan. Membuat pengalaman browsing terasa lebih cepat.
1

Buat queue dengan parameter Burst untuk PC1:

RouterOS CLI 
/queue simple add name=PC1-Burst target=192.168.1.10 max-limit=1M/2M burst-limit=2M/4M burst-threshold=512k/1M burst-time=10s/10s comment="PC1 with Burst"
2

Uji perilaku Burst:

Download file besar dari PC1. Perhatikan:

  • Kecepatan awal download → sesuai burst-limit (2Mbps/4Mbps)
  • Setelah beberapa detik → kecepatan turun ke max-limit normal (1Mbps/2Mbps)
⚠️
Burst aktif hanya ketika rata-rata penggunaan bandwidth di bawah burst-threshold. Jika pengguna terus-menerus mengunduh, burst tidak akan aktif kembali sampai penggunaan turun.

6 Praktikum D: Queue Tree

ℹ️
Queue Tree adalah metode QoS yang lebih canggih dibandingkan Simple Queue. Queue Tree menggunakan sistem parent-child (hierarki) dan bergantung pada MangleFitur MikroTik untuk menandai (marking) paket berdasarkan aturan tertentu untuk menandai paket terlebih dahulu. Cocok untuk jaringan kompleks yang membutuhkan prioritas berbeda untuk tiap jenis traffic.
📖 Konsep Queue Tree vs Simple Queue ▼
Aspek Simple Queue Queue Tree
Kemudahan konfigurasi ✅ Mudah ⚠️ Lebih kompleks
Hierarki / Parent-Child ❌ Tidak ada ✅ Ada (fleksibel)
Memerlukan Mangle ❌ Tidak ✅ Ya
Prioritas per traffic Terbatas ✅ Sangat fleksibel
Use case Limit per IP sederhana Prioritas VoIP, video, data berbeda

Cara kerja Queue Tree:

  1. Mangle menandai (mark) paket berdasarkan kriteria (IP, protokol, port, dll.)
  2. Queue Tree menangkap paket berdasarkan tanda (mark) tersebut
  3. Hierarchy parent mengatur total bandwidth, child mengatur prioritas dan porsi
1

Langkah 1: Buat Mangle untuk menandai paket (traffic marking)

Contoh: tandai traffic download dari internet (in-interface ether1 = WAN) dan upload ke internet.

RouterOS CLI 
/ip firewall mangle
add chain=forward action=mark-packet in-interface=ether1 \
    new-packet-mark=download-mark passthrough=no \
    comment="Tandai traffic download dari WAN"
add chain=forward action=mark-packet out-interface=ether1 \
    new-packet-mark=upload-mark passthrough=no \
    comment="Tandai traffic upload ke WAN"
2

Langkah 2: Buat Parent Queue (total bandwidth)

Parent Queue mengatur total bandwidth yang tersedia untuk internet.

RouterOS CLI 
/queue tree
add name=Total-Download parent=ether2 max-limit=10M \
    comment="Parent: total bandwidth download ke LAN"
add name=Total-Upload parent=ether1 max-limit=10M \
    comment="Parent: total bandwidth upload ke WAN"
3

Langkah 3: Buat Child Queue dengan prioritas berbeda

Child Queue digunakan untuk memberikan prioritas berbeda kepada jenis traffic yang berbeda. Prioritas 1 = tertinggi, 8 = terendah.

RouterOS CLI 
/queue tree
# Child Download dengan prioritas
add name=DL-Priority-1 parent=Total-Download \
    packet-mark=download-mark priority=1 limit-at=3M max-limit=10M \
    comment="Download prioritas tinggi (VoIP, video call)"

# Atau tanpa Mangle, gunakan PCQ sebagai child queue type:
add name=DL-All parent=Total-Download \
    packet-mark=download-mark queue=PCQ-Download max-limit=10M \
    comment="Download semua user - PCQ adil"
💡
Tips: limit-at = bandwidth minimum yang dijamin untuk child ini. max-limit = batas maksimum yang bisa dipakai jika parent masih punya bandwidth sisa.
4

Langkah 4: Verifikasi Queue Tree

RouterOS CLI 
/queue tree print
# Tampil daftar parent dan child queue
# Kolom "bytes" menunjukkan traffic yang sudah diproses

# Atau di Winbox:
# Queues → Queue Tree → lihat grafik real-time
⚠️
Penting: Jika menggunakan Queue Tree, hapus atau nonaktifkan Simple Queue yang memiliki target sama agar tidak terjadi konflik. Keduanya akan saling berebut mengatur traffic yang sama.

7 Praktikum E: Monitoring Queue

ℹ️
Memantau penggunaan bandwidth secara real-time menggunakan fitur monitoring MikroTik.
1

Pantau bandwidth real-time di Winbox:

Buka Queues → Simple Queues → klik dua kali pada queue → pilih tab Traffic. Grafik akan menampilkan penggunaan bandwidth upload dan download secara real-time.

2

Pantau dengan Torch (per-IP traffic):

RouterOS CLI 
/tool torch interface=ether2

Atau di Winbox: Tools → Torch, pilih interface ether2.

3

Bandwidth Test antar perangkat:

RouterOS CLI 
/tool bandwidth-test address=192.168.1.10 direction=both

8 Rubrik Penilaian

1
Simple Queue per user — Limit bandwidth per IP terverifikasi dengan speedtest
2
PCQ — Bandwidth dibagi rata saat multi-user aktif (diverifikasi dengan download bersamaan)
3
Monitoring bandwidth — Grafik traffic queue dan bandwidth test berhasil dilakukan
4
Konsep QoS — Mampu menjelaskan bandwidth, latency, jitter, dan perbedaan Simple Queue vs PCQ
