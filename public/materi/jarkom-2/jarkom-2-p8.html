1 Tujuan & Skenario
Tujuan Pembelajaran:
  1. Mengintegrasikan seluruh materi Jaringan Komputer 2 dalam satu implementasi nyata
  2. Merancang dan mengimplementasikan jaringan multi-gedung dengan berbagai layanan
  3. Memecahkan masalah jaringan kompleks secara sistematis
  4. Mendokumentasikan proses implementasi secara profesional

📋 Skenario: Jaringan Politeknik ABC

Politeknik ABC memiliki dua gedung yang perlu dihubungkan secara jaringan. Anda ditugaskan sebagai Network Engineer untuk merancang dan mengimplementasikan jaringan lengkap.

🏢 Gedung Utama — Kebutuhan Jaringan ▼
  • LAN Staf Admin (192.168.10.0/24) — 50 pengguna, DHCP, akses internet penuh
  • LAN Lab Komputer (192.168.20.0/24) — 30 PC, IP statis, akses terbatas
  • WiFi Mahasiswa (192.168.30.0/24) — SSID "Politala-Student", WPA2, DHCP, bandwidth 1M/user
  • Firewall — blokir sosmed jam 08:00-16:00, izinkan semua established
  • NAT Masquerade — semua LAN akses internet via 1 IP publik
  • QoS — Staf: 5Mbps priority, Lab: 2Mbps, Mahasiswa: 1Mbps/user (PCQ)
🏗️ Gedung Cabang — Kebutuhan Jaringan ▼
  • LAN Cabang (192.168.40.0/24) — 20 pengguna, DHCP
  • VPN Site-to-Site ke Gedung Utama via PPTP
  • Akses ke server di Gedung Utama (192.168.10.x) melalui VPN
🌐 Remote Access VPN ▼
  • Dosen/Staf bisa mengakses jaringan dari rumah via PPTP VPN
  • Pool IP VPN: 10.10.10.10-50
  • Akses ke server file (192.168.10.5) dan sistem akademik (192.168.10.6)

13 Panduan Penulisan Laporan
📄 Struktur Laporan Praktikum ▼
  1. Halaman Judul — Nama, NIM, Kelas, Tanggal, Mata Kuliah
  2. Tujuan Praktikum — Salin dari modul, tambahkan pemahaman sendiri
  3. Dasar Teori — Penjelasan singkat setiap teknologi yang digunakan (min. 2 halaman)
  4. Topologi Jaringan — Gambar topologi lengkap dengan addressing table
  5. Langkah Kerja — Setiap implementasi (A-G) dengan screenshot perintah dan hasilnya
  6. Hasil Pengujian — Tabel hasil ping, speedtest, dan verifikasi setiap komponen
  7. Analisis — Mengapa konfigurasi tersebut dipilih, masalah yang ditemui dan solusinya
  8. Kesimpulan — Rangkuman implementasi dan pembelajaran yang didapat
  9. Daftar Pustaka — Minimal 3 referensi yang relevan
📸 Screenshot yang Wajib Ada ▼
  • Output /ip address print Router Utama
  • Output /ip dhcp-server lease print — ada klien yang mendapat IP
  • Output /ip firewall filter print — semua rule firewall
  • Output /ip firewall nat print stats — NAT dengan counter > 0
  • Screenshot WiFi tersambung di smartphone/laptop
  • Output ping antar gedung melalui VPN (berhasil)
  • Speedtest dari PC di setiap subnet — menunjukkan bandwidth sesuai QoS
  • Output /queue simple print stats — counter menunjukkan traffic
