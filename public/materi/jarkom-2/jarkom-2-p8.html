1 Tujuan & Skenario
Tujuan Pembelajaran:
  1. Mengintegrasikan seluruh materi Jaringan Komputer 2 dalam satu implementasi nyata
  2. Merancang dan mengimplementasikan jaringan multi-gedung dengan berbagai layanan
  3. Memecahkan masalah jaringan kompleks secara sistematis
  4. Mendokumentasikan proses implementasi secara profesional

📋 Skenario: Jaringan Politeknik ABC

Politeknik ABC memiliki dua gedung yang perlu dihubungkan secara jaringan. Anda ditugaskan sebagai Network Engineer untuk merancang dan mengimplementasikan jaringan lengkap.

🏢 Gedung Utama — Kebutuhan Jaringan ▼
  • LAN Staf Admin (192.168.10.0/24) — 50 pengguna, DHCP, akses internet penuh
  • LAN Lab Komputer (192.168.20.0/24) — 30 PC, IP statis, akses terbatas
  • WiFi Mahasiswa (192.168.30.0/24) — SSID "Politala-Student", WPA2, DHCP, bandwidth 1M/user
  • Firewall — blokir sosmed jam 08:00-16:00, izinkan semua established
  • NAT Masquerade — semua LAN akses internet via 1 IP publik
  • QoS — Staf: 5Mbps priority, Lab: 2Mbps, Mahasiswa: 1Mbps/user (PCQ)
🏗️ Gedung Cabang — Kebutuhan Jaringan ▼
  • LAN Cabang (192.168.40.0/24) — 20 pengguna, DHCP
  • VPN Site-to-Site ke Gedung Utama via PPTP
  • Akses ke server di Gedung Utama (192.168.10.x) melalui VPN
🌐 Remote Access VPN ▼
  • Dosen/Staf bisa mengakses jaringan dari rumah via PPTP VPN
  • Pool IP VPN: 10.10.10.10-50
  • Akses ke server file (192.168.10.5) dan sistem akademik (192.168.10.6)

4 Implementasi A: Konfigurasi IP & Routing Dasar
1
/ip address add address=203.0.113.1/30 interface=ether1 comment="WAN-ISP" /ip address add address=192.168.10.1/24 interface=ether2 comment="LAN-Admin" /ip address add address=192.168.20.1/24 interface=ether3 comment="LAN-Lab" /ip address add address=192.168.30.1/24 interface=wlan1 comment="WiFi-Mahasiswa" /ip address print
2
/ip route add dst-address=0.0.0.0/0 gateway=203.0.113.2 /ip dns set servers=8.8.8.8,8.8.4.4 allow-remote-requests=yes /ip dns print
3
# Di Router Cabang /ip address add address=203.0.114.1/30 interface=ether1 comment="WAN-ISP-Cabang" /ip address add address=192.168.40.1/24 interface=ether2 comment="LAN-Cabang" /ip route add dst-address=0.0.0.0/0 gateway=203.0.114.2 /ip dns set servers=8.8.8.8,8.8.4.4
5 Implementasi B: DHCP Server (3 LAN)
1
/ip pool add name="Pool-Admin" ranges=192.168.10.100-192.168.10.200
2
/ip dhcp-server add \ name="DHCP-Admin" \ interface=ether2 \ address-pool=Pool-Admin \ lease-time=1d \ disabled=no /ip dhcp-server network add \ address=192.168.10.0/24 \ gateway=192.168.10.1 \ dns-server=8.8.8.8,192.168.10.1 \ comment="Network Admin"
/ip pool add name="Pool-WiFi" ranges=192.168.30.10-192.168.30.250 /ip dhcp-server add \ name="DHCP-WiFi" \ interface=wlan1 \ address-pool=Pool-WiFi \ lease-time=8h \ disabled=no /ip dhcp-server network add \ address=192.168.30.0/24 \ gateway=192.168.30.1 \ dns-server=8.8.8.8 \ comment="Network WiFi Mahasiswa"
# Di Router Cabang /ip pool add name="Pool-Cabang" ranges=192.168.40.100-192.168.40.200 /ip dhcp-server add \ name="DHCP-Cabang" \ interface=ether2 \ address-pool=Pool-Cabang \ lease-time=1d \ disabled=no /ip dhcp-server network add \ address=192.168.40.0/24 \ gateway=192.168.40.1 \ dns-server=8.8.8.8
6 Implementasi C: NAT Masquerade
1
/ip firewall nat add \ chain=srcnat \ out-interface=ether1 \ action=masquerade \ comment="NAT semua LAN ke internet"
2
# Di Router Cabang /ip firewall nat add \ chain=srcnat \ out-interface=ether1 \ action=masquerade \ comment="NAT Cabang ke internet"
7 Implementasi D: Firewall
⚠️ Urutan rule firewall sangat penting! Rule diproses dari atas ke bawah. Selalu letakkan rule established/related di paling atas.
1
/ip firewall filter add chain=input connection-state=established,related action=accept /ip firewall filter add chain=forward connection-state=established,related action=accept /ip firewall filter add chain=input connection-state=invalid action=drop /ip firewall filter add chain=forward connection-state=invalid action=drop
2
# Buat address list untuk IP sosmed (contoh) /ip firewall address-list add list=sosmed address=31.13.64.0/18 comment="Facebook" /ip firewall address-list add list=sosmed address=157.240.0.0/17 comment="Instagram" /ip firewall address-list add list=sosmed address=104.244.42.0/24 comment="Twitter/X" # Blokir sosmed dari WiFi mahasiswa /ip firewall filter add \ chain=forward \ src-address=192.168.30.0/24 \ dst-address-list=sosmed \ action=drop \ comment="Blokir sosmed dari WiFi"
3
/ip firewall filter add \ chain=input \ in-interface=ether1 \ dst-port=8291,22,23,80 \ protocol=tcp \ action=drop \ comment="Block router access dari WAN"
4
/ip firewall filter add chain=input protocol=icmp action=accept comment="Allow ICMP" /ip firewall filter add chain=forward protocol=icmp action=accept comment="Allow ICMP forward"
8 Implementasi E: Wireless AP WPA2
1
/interface wireless security-profiles add \ name="Politala-WPA2" \ mode=dynamic-keys \ authentication-types=wpa2-psk \ unicast-ciphers=aes-ccm \ group-ciphers=aes-ccm \ wpa2-pre-shared-key="PolitalaStudent2024!"
2
/interface wireless set wlan1 \ mode=ap-bridge \ ssid="Politala-Student" \ frequency=2437 \ band=2ghz-b/g/n \ channel-width=20mhz \ security-profile=Politala-WPA2 \ hide-ssid=no \ disabled=no
9 Implementasi F: QoS (Quality of Service)
1
/queue simple add \ name="QoS-Admin" \ target=192.168.10.0/24 \ max-limit=5M/5M \ limit-at=2M/2M \ priority=1 \ comment="Bandwidth Staf Admin"
2
/queue simple add \ name="QoS-Lab" \ target=192.168.20.0/24 \ max-limit=2M/2M \ limit-at=512k/512k \ priority=4 \ comment="Bandwidth Lab Komputer"
3
/queue type add \ name="PCQ-Student-DL" kind=pcq pcq-classifier=dst-address pcq-rate=1M /queue type add \ name="PCQ-Student-UL" kind=pcq pcq-classifier=src-address pcq-rate=512k /queue simple add \ name="QoS-WiFi" \ target=192.168.30.0/24 \ max-limit=20M/10M \ queue=PCQ-Student-UL/PCQ-Student-DL \ priority=8 \ comment="PCQ WiFi Mahasiswa"
10 Implementasi G: VPN PPTP Site-to-Site
1
/ip pool add name="VPN-Pool" ranges=10.10.10.10-10.10.10.50 /ppp profile add \ name="VPN-Profile" \ local-address=10.10.10.1 \ remote-address=VPN-Pool \ use-encryption=yes \ dns-server=8.8.8.8
2
/ppp secret add name="cabang" password="CabangPass123" service=pptp profile=VPN-Profile comment="Site-to-Site Cabang" /ppp secret add name="dosen1" password="Dosen1Pass" service=pptp profile=VPN-Profile comment="Remote Access Dosen" /ppp secret add name="staff1" password="Staff1Pass" service=pptp profile=VPN-Profile comment="Remote Access Staf"
3
/interface pptp-server server set enabled=yes default-profile=VPN-Profile authentication=mschap2 /ip firewall filter add chain=input protocol=tcp dst-port=1723 action=accept comment="Allow PPTP" /ip firewall filter add chain=input protocol=gre action=accept comment="Allow GRE"
1
# Di Router Cabang /interface pptp-client add \ name="VPN-HQ" \ connect-to=203.0.113.1 \ user=cabang \ password="CabangPass123" \ add-default-route=no \ disabled=no
2
/ip route add dst-address=192.168.10.0/24 gateway=VPN-HQ comment="Route ke LAN Admin HQ" /ip route add dst-address=192.168.20.0/24 gateway=VPN-HQ comment="Route ke LAN Lab HQ"
3
# Kembali di Router Utama /ip route add dst-address=192.168.40.0/24 gateway=10.10.10.1 comment="Route ke LAN Cabang via VPN"
💡 Untuk Remote Access, dosen/staf cukup mengkonfigurasi VPN client di Windows/macOS menggunakan username dan password yang telah dibuat di PPP Secrets.
Konfigurasi VPN Windows: - Server: 203.0.113.1 - Tipe: PPTP - Username: dosen1 / staff1 - Password: sesuai yang dikonfigurasi - Setelah terhubung, bisa akses 192.168.10.5 (File Server) dan 192.168.10.6 (Sistem Akademik)

13 Panduan Penulisan Laporan
📄 Struktur Laporan Praktikum ▼
  1. Halaman Judul — Nama, NIM, Kelas, Tanggal, Mata Kuliah
  2. Tujuan Praktikum — Salin dari modul, tambahkan pemahaman sendiri
  3. Dasar Teori — Penjelasan singkat setiap teknologi yang digunakan (min. 2 halaman)
  4. Topologi Jaringan — Gambar topologi lengkap dengan addressing table
  5. Langkah Kerja — Setiap implementasi (A-G) dengan screenshot perintah dan hasilnya
  6. Hasil Pengujian — Tabel hasil ping, speedtest, dan verifikasi setiap komponen
  7. Analisis — Mengapa konfigurasi tersebut dipilih, masalah yang ditemui dan solusinya
  8. Kesimpulan — Rangkuman implementasi dan pembelajaran yang didapat
  9. Daftar Pustaka — Minimal 3 referensi yang relevan

  • Output /ip dhcp-server lease print — ada klien yang mendapat IP
  • Output /ip firewall filter print — semua rule firewall
  • Output /ip firewall nat print stats — NAT dengan counter > 0

  • Output ping antar gedung melalui VPN (berhasil)
