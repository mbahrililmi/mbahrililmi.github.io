2 Teori VPN & Tunneling

Tunneling adalah teknik mengenkapsulasi protokol jaringan di dalam protokol lain, menciptakan "terowongan" virtual melalui jaringan publik. VPNVirtual Private Network: koneksi aman terenkripsi melalui internet publik menggunakan tunneling untuk memberikan koneksi privat yang terenkripsi melalui internet.

🏠 Remote Access VPN ▼

Memungkinkan pengguna individual (mobile/remote worker) terhubung ke jaringan kantor dari mana saja melalui internet. Contoh: karyawan WFH yang mengakses server internal.

  [Laptop WFH]──Internet──[VPN Server / MikroTik]──[Jaringan Kantor]
  IP: 10.10.10.2                IP: 203.x.x.x          192.168.1.0/24
🏢 Site-to-Site VPN ▼

Menghubungkan dua atau lebih jaringan (gedung/cabang) secara permanent melalui internet. Router di kedua sisi membentuk tunnel otomatis.

  [Gedung A]──[MikroTik A]──Internet──[MikroTik B]──[Gedung B]
  192.168.1.0/24  203.0.113.1     203.0.113.5  192.168.2.0/24
               ←─────── PPTP/L2TP Tunnel ───────→
📡 PPP (Point-to-Point Protocol) ▼

PPP adalah protokol data link layer yang menyediakan koneksi point-to-point. Mendukung autentikasi (PAP, CHAP, MS-CHAPv2), enkripsi, dan kompresi.

  • PAP: Password Authentication Protocol — password dikirim clear text (tidak aman)
  • CHAP: Challenge-Handshake Authentication Protocol — hash MD5 (lebih aman)
  • MS-CHAPv2: Microsoft CHAP v2 — digunakan PPTP/L2TP, mendukung MPPE
