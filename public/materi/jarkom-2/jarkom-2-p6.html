2 Teori VPN & Tunneling

Tunneling adalah teknik mengenkapsulasi protokol jaringan di dalam protokol lain, menciptakan "terowongan" virtual melalui jaringan publik. VPNVirtual Private Network: koneksi aman terenkripsi melalui internet publik menggunakan tunneling untuk memberikan koneksi privat yang terenkripsi melalui internet.

🏠 Remote Access VPN ▼

Memungkinkan pengguna individual (mobile/remote worker) terhubung ke jaringan kantor dari mana saja melalui internet. Contoh: karyawan WFH yang mengakses server internal.

  [Laptop WFH]──Internet──[VPN Server / MikroTik]──[Jaringan Kantor]
  IP: 10.10.10.2                IP: 203.x.x.x          192.168.1.0/24
🏢 Site-to-Site VPN ▼

Menghubungkan dua atau lebih jaringan (gedung/cabang) secara permanent melalui internet. Router di kedua sisi membentuk tunnel otomatis.

  [Gedung A]──[MikroTik A]──Internet──[MikroTik B]──[Gedung B]
  192.168.1.0/24  203.0.113.1     203.0.113.5  192.168.2.0/24
               ←─────── PPTP/L2TP Tunnel ───────→
📡 PPP (Point-to-Point Protocol) ▼

PPP adalah protokol data link layer yang menyediakan koneksi point-to-point. Mendukung autentikasi (PAP, CHAP, MS-CHAPv2), enkripsi, dan kompresi.

  • PAP: Password Authentication Protocol — password dikirim clear text (tidak aman)
  • CHAP: Challenge-Handshake Authentication Protocol — hash MD5 (lebih aman)
  • MS-CHAPv2: Microsoft CHAP v2 — digunakan PPTP/L2TP, mendukung MPPE

4 Lab: Konfigurasi IP Pool
1
/ip pool add \ name="VPN-Pool" \ ranges=10.10.10.10-10.10.10.50 /ip pool print
5 Lab: PPP Profile
1
/ppp profile add \ name="VPN-Profile" \ local-address=10.10.10.1 \ remote-address=VPN-Pool \ use-encryption=yes \ dns-server=8.8.8.8,8.8.4.4 \ only-one=yes \ comment="Profile untuk VPN clients" /ppp profile print
💡 local-address adalah IP VPN yang digunakan router (server side). remote-address adalah IP yang diberikan ke klien, bisa IP langsung atau nama Pool.
6 Lab: PPP Secrets (Akun VPN)
1
/ppp secret add \ name="user1" \ password="Password123!" \ service=pptp \ profile=VPN-Profile \ comment="Akun VPN User 1" /ppp secret add \ name="user2" \ password="SecretPass456!" \ service=any \ profile=VPN-Profile /ppp secret print
7 Lab: PPTP VPN Server
1
/interface pptp-server server set \ enabled=yes \ default-profile=VPN-Profile \ authentication=mschap2 \ mrru=1600
2
/ip firewall filter add \ chain=input \ protocol=tcp \ dst-port=1723 \ action=accept \ comment="Allow PPTP" /ip firewall filter add \ chain=input \ protocol=gre \ action=accept \ comment="Allow GRE (PPTP encapsulation)"
3
# Ini akan otomatis ketika klien terhubung # Verifikasi dengan: /ip route print
⚠️ PPTP dianggap tidak aman secara modern karena kelemahan dalam protokol MS-CHAPv2 dan MPPE. Untuk keamanan yang lebih baik, gunakan L2TP/IPsec atau WireGuard.
8 Lab: PPTP Client (MikroTik ke MikroTik)
1
/interface pptp-client add \ name="VPN-to-HQ" \ connect-to=203.0.113.10 \ user=user1 \ password="Password123!" \ profile=default \ add-default-route=no \ disabled=no
2
/ip route add \ dst-address=192.168.1.0/24 \ gateway=VPN-to-HQ \ comment="Route ke LAN pusat via VPN"
3
/interface pptp-client print /ping 192.168.1.1 # Ping ke LAN HQ melalui tunnel
9 Lab: L2TP/IPsec Server
1
/interface l2tp-server server set \ enabled=yes \ default-profile=VPN-Profile \ authentication=mschap2 \ use-ipsec=yes \ ipsec-secret="IPsecSharedKey123" \ mrru=1600
2
/ip firewall filter add \ chain=input protocol=udp \ dst-port=1701,500,4500 \ action=accept \ comment="Allow L2TP/IPsec" /ip firewall filter add \ chain=input protocol=ipsec-esp \ action=accept \ comment="Allow IPsec ESP"

11 Tugas
