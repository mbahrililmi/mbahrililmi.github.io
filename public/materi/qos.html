Pengertian QoS

Quality of Service (QoS) adalah seperangkat teknologi dan mekanisme untuk mengatur dan memprioritaskan lalu lintas jaringan agar sumber daya (bandwidth) digunakan secara optimal dan adil.

Tanpa QoS, satu pengguna bisa memonopoli seluruh bandwidth sehingga pengguna lain tidak kebagian koneksi yang layak.

4
Simple Queue

Cara Kerja

MikroTik mencocokkan setiap paket dengan daftar Simple Queue dari atas ke bawah. Begitu ada yang cocok, aturan bandwidth langsung diterapkan dan paket berikutnya tidak dicek lagi ke bawah.

Parameter Utama

Parameter Keterangan Contoh
Target IP address atau subnet yang dibatasi 192.168.1.10/32
Max Limit Batas bandwidth (format: upload/download) 1M/2M
Burst Limit Batas saat burst (lebih tinggi dari max-limit) 2M/4M
Burst Time Durasi burst diizinkan (detik) 10s/10s
Queue Type Jenis antrian (default / PCQ) default / PCQ-Upload
💡 Format Max Limit: upload/download — contoh 1M/2M = upload 1 Mbps, download 2 Mbps

Konfigurasi — Winbox

Buka Queues → Simple Queues → [+], isi parameter berikut:

# Contoh: Limit PC1 — upload 1M, download 2M Name : PC1-Limit Target : 192.168.1.10/32 Max Limit : Upload 1M / Download 2M Comment : Limit PC1 - 1M/2M

Konfigurasi — CLI (Terminal Winbox)

# PC1 /queue simple add name=PC1-Limit target=192.168.1.10 max-limit=1M/2M # PC2 /queue simple add name=PC2-Limit target=192.168.1.20 max-limit=512k/1M # PC3 /queue simple add name=PC3-Limit target=192.168.1.30 max-limit=2M/4M # PC4 /queue simple add name=PC4-Limit target=192.168.1.40 max-limit=1M/2M

Verifikasi: buka speedtest.net dari PC1 → hasil harus ≤ 2 Mbps download, ≤ 1 Mbps upload.

PCQ — Per Connection Queue

Prinsip Kerja PCQ

PCQ adalah Queue Type khusus yang membagi bandwidth secara adil kepada semua pengguna aktif secara otomatis. Tidak perlu konfigurasi per IP — cukup satu aturan untuk seluruh jaringan.

1 Pengguna → 10 Mbps masing-masing
2 Pengguna → 5 Mbps masing-masing
4 Pengguna → 2.5 Mbps masing-masing
8 Pengguna → 1.25 Mbps masing-masing

🔢 Simulasi PCQ — Geser untuk melihat pembagian

2

10 Mbps
Per Pengguna
5 Mbps
Saat pengguna baru aktif, PCQ otomatis menyesuaikan jatah bandwidth. Saat pengguna tidak aktif, bandwidth-nya diberikan ke yang lain.

PCQ Classifier

Classifier Digunakan untuk Alasan
dst-address Download Membedakan pengguna berdasarkan alamat tujuan paket
src-address Upload Membedakan pengguna berdasarkan alamat sumber paket

Konfigurasi PCQ — Langkah Demi Langkah

Buat dua Queue Type terlebih dahulu, kemudian buat satu Simple Queue yang menggunakannya.

Langkah 1 — Buat Queue Type PCQ Download:

# Queues → Queue Types → [+] Name : PCQ-Download Kind : pcq PCQ Rate : 0 (bagi rata otomatis) PCQ Classifier : dst-address # Via CLI: /queue type add name=PCQ-Download kind=pcq pcq-rate=0 pcq-classifier=dst-address

Langkah 2 — Buat Queue Type PCQ Upload:

/queue type add name=PCQ-Upload kind=pcq pcq-rate=0 pcq-classifier=src-address

Langkah 3 — Buat Simple Queue menggunakan PCQ:

# Queues → Simple Queues → [+] Name : Internet-PCQ Target : 192.168.1.0/24 Max Limit Upload : 10M Max Limit Download : 10M Queue Type Upload : PCQ-Upload Queue Type Download: PCQ-Download # Via CLI: /queue simple add name=Internet-PCQ target=192.168.1.0/24 \ max-limit=10M/10M queue=PCQ-Upload/PCQ-Download
6
Burst

Burst mengizinkan pengguna untuk sementara melebihi batas bandwidth, kemudian kembali ke batas normal. Berguna agar halaman web terasa cepat saat pertama dimuat.

Parameter Burst

Parameter Keterangan Contoh (Upload / Download)
max-limit Batas bandwidth normal 1M / 2M
burst-limit Batas saat burst (lebih tinggi) 2M / 4M
burst-threshold Trigger: burst aktif jika rata-rata di bawah nilai ini 512k / 1M
burst-time Durasi burst diizinkan (detik) 10s / 10s

Konfigurasi Burst via CLI

/queue simple add name=PC1-Burst target=192.168.1.10 \ max-limit=1M/2M \ burst-limit=2M/4M \ burst-threshold=512k/1M \ burst-time=10s/10s

Kesimpulan

QoS secara umum

QoS adalah mekanisme kendali bandwidth agar jaringan digunakan secara adil dan efisien. Tanpa QoS, pengguna dengan aktivitas berat (streaming, download besar) bisa memonopoli seluruh bandwidth.

Simple Queue

Paling mudah dikonfigurasi. Cocok untuk membatasi bandwidth per pengguna secara manual. Kelemahan: tidak skalabel — harus dibuat satu aturan per IP pengguna.

PCQ (Per Connection Queue)

Solusi cerdas untuk jaringan multi-pengguna. Satu aturan untuk seluruh subnet, bandwidth dibagi otomatis dan proporsional. Bila ada pengguna tidak aktif, jatahnya diberikan ke pengguna lain sehingga bandwidth tidak terbuang.

Burst

Bukan mekanisme terpisah, melainkan fitur tambahan pada Simple Queue. Mengizinkan kecepatan lebih tinggi secara sementara sebelum kembali ke batas normal. Berguna untuk mempercepat loading awal (web, email).

Lima Parameter QoS

  • Bandwidth — kapasitas maksimum jalur
  • Throughput — bandwidth aktual yang terkirim
  • Latency — waktu tempuh paket (ideal <150ms untuk VoIP)
  • Jitter — variasi keterlambatan (ideal <30ms untuk VoIP)
  • Packet Loss — paket yang hilang (ideal <1% untuk VoIP)
